# 如何拿到并运行「桌面待办」（新手零基础版）

这是你和 AI 一起做的第一个应用。下面按**你选的路线**来：你选了「**云端自动编译、自动出 .exe 安装包**」，所以不用在自己电脑上装 Rust / VS 那一套，只要会"点下载、点安装"就行。

---

## 方式一（推荐，你正在用的）：云端编译 → Gitee 下载安装

你不需要装任何开发软件，也不用会编程。

### 第 1 步：让 AI 触发一次云端构建
> 你只要对 AI 说一句，例如：「帮我出个新版本 / 构建一下 / 重新打包」。AI 会在 **GitHub 云端**（Windows 服务器）把代码编译成安装包，整个过程约 **10 分钟**。

### 第 2 步：去 Gitee 发行版下载
1. 浏览器打开 **https://gitee.com/minting2/desktop-todo/releases**
2. 页面**最上方（最新）**的那一条发行版，点开它
3. 在附件里下载 **`desktoptodo-setup-x64.exe`**（约 1.8 MB）

> 💡 为什么要点页面里的按钮，而不是一个直链？因为这个仓库是**私有**的，下载要在你**登录 Gitee 后**的"发行版"页面里点才有效（直接复制链接会被拦 403）。登录后点一下就行，很方便。GitHub 那边也会同步存一份（兜底），但走 Gitee 最省事。

### 第 3 步：安装并运行
1. 双击 `desktoptodo-setup-x64.exe` → 一路「下一步」安装
2. 安装完会自动（或手动从开始菜单）打开，**桌面上贴出一个透明、置顶的小窗口** → 成功 🎉

> ⚠️ 第一次安装 Windows 可能会拦截（SmartScreen 提示"已保护你的电脑"）。这是正常的，点「详细信息 → 仍要运行」即可，**不用花钱买证书、不影响使用**。

### 你能对这个窗口做什么
- 鼠标按**顶部栏**拖动窗口
- 鼠标移到窗口**四边或四角**，光标变双向箭头时按住拖动 → 调整大小（你确认过的原型效果）
- 点列表**空白处**直接输入待办；点圆圈打勾；点 🔧 编辑详情；拖左侧 `⋮⋮` 排序；右上角 ⚙ 是设置（含窗口透明度）

## 方式二（可选 / 进阶）：在自己电脑本地编译

> 只有当你想**自己改代码、自己马上编译看效果**、不想等云端时，才需要走这条路。首次要装 4 个免费软件，约 20~40 分钟。

### 总览：你要做 4 件事
1. **拿到代码**（从 Gitee 下载 ZIP，解压）
2. **装 4 个免费软件**（Node、pnpm、Rust、VS 生成工具）
3. **运行一行命令**让透明窗口弹出来
4. （可选）打包成安装程序

### 怎么打开"命令行"（后面会反复用到）
1. 按 `Win` 键 → 输入 `PowerShell`（或 `终端` / `Terminal`）
2. 点搜索结果里的 **「Windows PowerShell」**
3. 弹出蓝底（或黑底）窗口，光标在闪 → 这就是命令行（后面说的"输入 XXX"都是在这里右键粘贴后回车）

### 第 1 步：拿到代码
1. 访问 **https://gitee.com/minting2/desktop-todo** → 右上角绿色 **「克隆 / 下载」** → **「下载 ZIP」**
2. 解压，把文件夹**移到 D 盘根目录**并重命名为 `desktop-todo`（最终 `D:\desktop-todo`，**不含中文、不含空格**）

### 第 2 步：安装 4 个免费软件（按顺序）
- **2.1 Node.js**：访问 **https://nodejs.org** → 点首页绿色 **「LTS」** → 下载 `.msi` → 一路「下一步」。验证：命令行输入 `node -v` 看到版本号。
- **2.2 pnpm**：命令行输入 `npm install -g pnpm`，验证 `pnpm -v`。
  > 若报"禁止运行脚本"，先跑 `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned`（选 Y），再重试。
- **2.3 Rust**：访问 **https://rustup.rs** → 点 **「Download rustup-init.exe (64-bit)」** 下载双击 → 黑窗口里**直接回车**选默认 → 装完验证 `cargo --version`。
- **2.4 Visual Studio 生成工具**（最容易错）：访问 **https://visualstudio.microsoft.com/zh-hans/downloads/** → 往下滚到「Visual Studio 2022 工具」→ 下载 **「Visual Studio 2022 生成工具 / Build Tools」** → 安装时**只勾「使用 C++ 的桌面开发」** → 安装（10~20 分钟）。
- **2.5 WebView2 运行时**：Win11/较新 Win10 一般已自带；「添加或删除程序」里搜 WebView2 能看到即可，没有就去微软官网下「Evergreen Bootstrapper」。

### 第 3 步：让应用跑起来
```
cd D:\desktop-todo
pnpm install
pnpm tauri dev
```
第一次会编译 Rust（几分钟，黑窗口刷字正常），完事自动弹出透明窗口 → 成功 🎉

### 第 4 步：本地打包安装程序
```
pnpm tauri build
```
产物在 `D:\desktop-todo\src-tauri\target\release\bundle\`，里面有 `.exe` 安装程序。

## 常见问题
**Q：Gitee 发行版里那条下载下来是空文件 / 打不开？**
→ 确认是在**登录 Gitee 后**点发行版里的下载按钮，且下的是 `desktoptodo-setup-x64.exe`（不是 `build-x.zip` 那种源码压缩包）。

**Q：命令提示"不是内部或外部命令"？**（仅方式二）
→ 软件没装好，或装完没**重开命令行窗口**。重开再试；仍不行说明该项没装上，回去第 2 步补装。

**Q：`pnpm tauri dev` 卡住/报错？**（仅方式二）
→ 第一次编译 Rust 要几分钟，耐心等。若报 LINK / MSVC 相关错误，是 2.4 的 VS 生成工具没装或没勾「使用 C++ 的桌面开发」，回去补装。

**Q：窗口是白的/不透明？**
→ 正常应是透明。看不清就打开 ⚙ 设置调低"窗口透明度"。若完全纯白，可能是 WebView2（2.5）缺失，去装一下。

**Q：改了代码怎么看效果？**（仅方式二）
→ `pnpm tauri dev` 跑着时，改 `src/` 保存自动刷新；改 `src-tauri/` 的 Rust 代码需停掉（Ctrl+C）重跑。
