# 在你自己的 Windows 电脑上跑起来（新手零基础版）

这是你用 AI 写出的第一个应用。下面每一步都很详细，照着做就行。**不需要你会编程**，只需要会点"下一步"、会复制粘贴几行命令。

---

## 总览：你要做 4 件事

1. **拿到代码**（从 Gitee 下载 ZIP，解压）
2. **装 4 个免费软件**（Node、pnpm、Rust、VS 生成工具）—— 这是你正在做的这一步
3. **运行一行命令**让透明窗口弹出来
4. （可选）打包成安装程序发给朋友

预计耗时：首次安装约 20~40 分钟（主要在下载和 Rust 首次编译），之后每次运行只要几秒。

---

## 怎么打开"命令行"（后面会反复用到）

本文里很多步骤要你在"命令行窗口"里输入内容。打开方法：

1. 按键盘左下角带 **Windows 图标**的 `Win` 键
2. 输入 `PowerShell`（或 `终端` / `Terminal`）
3. 在搜索结果里点 **「Windows PowerShell」**（或「终端」）
4. 会弹出一个**蓝底（或黑底）的窗口**，光标在闪 —— 这就是命令行

> 后面说的"在命令行输入 XXX"，都是指在这个窗口里**右键粘贴**或**手动输入**那行字，然后按 `回车` 键。

## 第 1 步：拿到代码（从 Gitee 下载，不用装 git）

1. 打开浏览器访问 **https://gitee.com/minting2/desktop-todo**
2. 点页面右上角绿色的 **「克隆 / 下载」** 按钮 → 再点 **「下载 ZIP」**
3. 浏览器会把 `desktop-todo-master.zip` 下载到你的「下载」文件夹
4. 找到这个 zip → 右键 →「解压到 desktop-todo-master」（或双击解压）
5. 把解压出来的文件夹**移动到 D 盘根目录**，并重命名为 `desktop-todo`
   - 最终路径应是 `D:\desktop-todo`（**不含中文、不含空格**）
6. 打开看看里面应有：`src`、`src-tauri`、`package.json`、`README.md`、`docs/` 等

> ⚠️ 路径不要有中文和空格，否则后面命令会报错。例如 `D:\我的待办` 不行，`D:\desktop-todo` 可以。

## 第 2 步：安装 4 个免费软件（按顺序来，别跳）

### 2.1 Node.js（必装，最基础的运行环境）

**它是什么**：项目能跑起来的底层环境，装一次永久用。

1. 浏览器访问 **https://nodejs.org**（国内能直接打开，不用 VPN）
2. 首页正中间有两个大按钮：
   - 左边绿色 **「LTS」**（长期支持版，推荐）→ **点它**
   - 右边「Current」不用管
3. 浏览器自动下载一个 `.msi` 安装文件（文件名类似 `node-v20.18.0-x64.msi`）
4. 双击它 → 一路点 **「Next / 下一步」**，**所有选项都用默认**，最后点「Install / 安装」
5. 提示完成后关掉窗口

**验证装好没（很重要）**：
- 打开一个命令行窗口（方法见上方「怎么打开命令行」）
- 输入：
  ```
  node -v
  ```
- 看到类似 `v20.18.0` 的版本号 → ✅ 成功
- 如果提示"node 不是内部或外部命令"：**把这个命令行窗口关掉重新打开**再试

---

### 2.2 pnpm（必装，包管理器）

**它是什么**：一条命令帮我们把项目需要的依赖一次性装好。

在刚才的命令行窗口里输入（复制粘贴这行，回车）：
```
npm install -g pnpm
```
等十几秒，没报错就成。验证：
```
pnpm -v
```
看到版本号（如 `9.12.0`）→ ✅ 成功

> 💡 如果报错"无法加载 pnpm，因为在此系统上禁止运行脚本"：先执行这一句再重试：
> ```
> Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
> ```
> 弹出询问选「Y / 全是」，然后再跑上面的 `npm install -g pnpm`。

---

### 2.3 Rust（必装，把代码编译成 exe 的工具）

**它是什么**：Tauri 靠它把我们的代码真正"编译"成 Windows 程序。必须装。

1. 浏览器访问 **https://rustup.rs**
2. 页面上部有一段命令，下面有一行**链接**，找写着 **「Download rustup-init.exe (64-bit)」** 的那行 → **点它下载**
   - ⚠️ 不是复制上面那段命令，是点那个 `.exe` 下载链接
3. 双击下载的 `rustup-init.exe`
4. 弹出一个**黑色命令行窗口**，最后一行问你安装选项：
   ```
   1) Proceed with standard installation (default - just press enter)
   2) Customize installation
   3) Cancel installation
   >_
   ```
   **直接按回车键**（选默认 1 就行）
5. 自动下载（几分钟，窗口刷字是正常的），最后看到 `Rust is installed!` 字样
6. **关掉这个黑窗口**，重新打开一个命令行窗口，输入：
   ```
   cargo --version
   ```
   看到 `cargo 1.xx.x` → ✅ 成功

---

### 2.4 Visual Studio 生成工具（必装，C++ 编译环境）⚠️ 最容易装错

**它是什么**：Rust 编译 Windows 程序时，还需要微软的 C++ 底层工具链。这一步最容易点错，仔细看。

1. 浏览器访问 **https://visualstudio.microsoft.com/zh-hans/downloads/**
2. 页面很长，往**下滚动**，找到一块标题叫 **「Visual Studio 2022 工具」**（Tools 那一栏）
3. 在里面找 **「Visual Studio 2022 生成工具」**（英文名 **Build Tools for Visual Studio 2022**）→ 点它右边的 **「下载」**
   - ⚠️ **不是**「Visual Studio Community」（那是完整 IDE，几个 G，太大）；要的是「**生成工具 / Build Tools**」
4. 双击下载的 `vs_BuildTools.exe`，会先下载一个小安装器
5. 安装器出来后，中间有个 **「工作负载 / Workloads」** 标签页
6. 在列表里**只勾选一项：「使用 C++ 的桌面开发」**
   - 右侧"安装详细信息"里会自动带上「MSVC v143 ... 生成工具」和「Windows 10/11 SDK」，保持勾选即可
7. 右下角点 **「安装 / Install」**，等 10~20 分钟（要联网下载）
8. 装完点「关闭」。**你不需要打开 Visual Studio 写代码**，这个工具只是后台编译用

---

### 2.5 WebView2 运行时（一般已自带，检查即可）

**它是什么**：Tauri 窗口底层用的微软浏览器内核。Win11 和较新的 Win10 都预装了。

- **大概率你什么都不用做。**
- 检查方法：按 `Win` 键输入「添加或删除程序」，在列表里搜「WebView2」，能看到「Microsoft Edge WebView2 Runtime」就说明有了。
- 如果搜不到（很少见），访问微软官网搜「WebView2 Runtime 下载」，下载「常青版引导程序（Evergreen Bootstrapper）」双击安装即可。

## 第 3 步：让应用跑起来 🎉

1. 打开 **命令行窗口**
2. 进入你的项目目录（路径换成你解压的位置）：
   ```
   cd D:\desktop-todo
   ```
3. 安装前端依赖（第一次较慢，要下载一会儿）：
   ```
   pnpm install
   ```
4. **启动应用**（关键一步）：
   ```
   pnpm tauri dev
   ```
5. 第一次会**编译 Rust**（几分钟，黑窗口刷很多字是正常的），编译完后会自动弹出一个**透明的小窗口**贴在桌面上 → 成功！🎉

> 窗口是透明的、置顶的。鼠标按住顶部栏可以拖动窗口；**鼠标移到窗口四边或四角**，光标变成双向箭头时按住拖动，就能调整大小（这就是你确认的原型效果）；点击列表空白处直接输入待办；点圆圈打勾；点 🔧 编辑详情；拖左侧 `⋮⋮` 排序；右上角 ⚙ 是设置。

## 第 4 步：打包成安装程序（发给朋友）

应用做好后，想发给朋友：

```
pnpm tauri build
```

完成后，安装包在这里：
```
D:\desktop-todo\src-tauri\target\release\bundle\
```
里面有 `.msi` 或 `.exe` 安装程序，发给朋友双击安装即可。

> 朋友首次安装会被 Windows 拦截（SmartScreen 提示"已保护你的电脑"），点"详细信息 → 仍要运行"就行。这是正常的，不影响使用，不用花钱买证书。

## 常见问题

**Q：命令提示"不是内部或外部命令"？**
→ 软件没装好，或装完后没**重开命令行窗口**。重开再试；仍不行说明该软件没装上，回去第 2 步重装那一项。

**Q：`pnpm tauri dev` 卡住/报错？**
→ 第一次编译 Rust 要几分钟，耐心等。若报 "LINK" / "MSVC" 相关错误，通常是 2.4 的 VS 生成工具没装，或没勾「使用 C++ 的桌面开发」，回去补装。

**Q：窗口是白的/不透明？**
→ 正常应是透明。若你系统主题导致看不清，打开 ⚙ 设置调低"窗口透明度"。若完全不透明且是纯白，可能是 WebView2（2.5）缺失，去装一下。

**Q：改了代码怎么看效果？**
→ `pnpm tauri dev` 在跑时，改 `src/` 下的文件保存后会自动刷新。改 `src-tauri/` 下的 Rust 代码需要停掉（Ctrl+C）再重新 `pnpm tauri dev`。

**Q：我不想装这么多东西，只想先看界面？**
→ 可以！只装 2.1 和 2.2（Node + pnpm），然后在项目目录运行：
> ```
> pnpm dev
> ```
> 浏览器打开 http://localhost:1420 就能看界面（用浏览器 localStorage 存数据，没有透明窗口效果）。
